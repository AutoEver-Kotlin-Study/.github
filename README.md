- **요구사항은 자체적으로 정의하여 실습을 진행합니다.**
  - 요구사항을 개인적으로 추가 및 수정을 허용하는 것도 허용합니다. 단, README 파일에 해당 내용을 기록합니다.

## 📌 주제 및 요구사항

### 그룹 위치 공유 시나리오
- 사용자는 그룹을 생성하거나 방번호로 참여할 수 있다.
- 사용자는 여러 그룹에 속할 수 있다.
- 그룹 생성시 최대 참여 인원을 설정할 수 있다. (예: 최대 10명)
- 새로운 참여자 추가 혹은 퇴장시 기존 그룹에 속한 인원들에게 SMS 알림을 발송할 수 있다.
- 사용자는 자신의 현재 위치(위도/경도)를 업데이트할 수 있다.
- 참여자는 그룹내 다른 인원들의 위치를 조회할 수 있다.
- 그룹 생성자는 그룹을 삭제할 수 있다.
- 참여자는 방을 퇴장할 수 있다.

## 🔌 API 목록

| 기능 | Method | Endpoint |
|------|--------|----------|
| 그룹 생성 API | `POST` | `/groups` |
| 소속된 그룹 목록 조회 API | `GET` | `/groups` |
| 그룹 참여 API | `POST` | `/groups/{roomCode}/join` |
| 그룹 위치 조회 API | `GET` | `/groups/{groupId}/locations` |
| 그룹 위치 업데이트 API | `PUT` | `/groups/{groupId}/location` |
| (참여자) 그룹 퇴장 API | `DELETE` | `/groups/{groupId}/leave` |
| 그룹 삭제 API | `DELETE` | `/groups/{groupId}` |

## 📲 SMS 서버 연동

SMS 알림은 로컬 SMS 목서버(`localhost:8090`)를 통해 발송됩니다.

**Endpoint:** `POST /sms/send`

```json
{
  "from": "010-0000-0000",
  "to": "010-1234-5678",
  "content": "OOO님이 그룹 [그룹명]에 참여하였습니다."
}
```

| 필드 | 설명 |
|------|------|
| `from` | 발신번호 (서비스 대표 번호, 고정값) |
| `to` | 수신번호 (그룹원 개인 번호) |
| `content` | 메시지 본문 |

> 참여자 입장 및 퇴장 시 해당 그룹의 모든 기존 멤버에게 SMS가 발송됩니다.
